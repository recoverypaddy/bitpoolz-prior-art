# DISCLOSURE 6: Real-World Event Triggered Prize Pool

## Technical Field
Prize pool systems whose draw timing is conditional on real-world event outcomes rather than pool fill completion.

## Background
Tying prize pool draws to real-world events (Bitcoin halving, sports results, price thresholds, election outcomes) creates inherent marketing moments and natural distribution opportunities.

## Detailed Description
A prize pool variant whose draw trigger is conditional on one or more real-world events occurring, verified through a trusted oracle. Event types supported: (a) Bitcoin protocol events — block height milestones, halving events, difficulty adjustments; (b) Price oracle events — BTC/USD price crossing a threshold (verified via multiple oracle sources with median price); (c) Sports/competition results — verified via signed data feeds from licensed sports data providers; (d) Time-based events — specific calendar dates/times. The system handles: event definition at pool creation (oracle source, event criteria, verification threshold), participant disclosure (clear display of event dependency and estimated timing), conditional draw trigger (pool fills normally but draw only executes when event condition is met), maximum wait period (if event does not occur within X days, participants receive full refund to Drip balance), multi-oracle consensus (minimum 3 oracle sources must agree on event outcome), and event verification recording in public ledger.

## Key Claims
- Event-conditional draw trigger mechanism
- Multiple oracle source types (Bitcoin protocol, price, sports, time)
- Multi-oracle consensus requirement (minimum 3 sources)
- Maximum wait period with automatic refund
- Event definition immutability after pool creation
- Event verification recording in public ledger
- Participant disclosure of event dependency at entry time

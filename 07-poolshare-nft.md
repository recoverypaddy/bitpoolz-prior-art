# DISCLOSURE 7: PoolShare Non-Fungible Token (NFT) for Verified Position History

## Technical Field
Non-fungible token representation of verified sweepstakes position ownership and draw outcome history.

## Background
A permanent, transferable, verifiable artifact of pool participation and winning creates collector value, social sharing incentive, and cross-platform interoperability for position history.

## Detailed Description
A non-fungible token minted on Bitcoin via the Ordinals protocol (or compatible Bitcoin layer) representing verified ownership of a specific ticket range in a BitPoolz pool at a specific point in time. Standard PoolShare NFT metadata: pool ID, pool name, pool tier, ticket range (start, end), sats entered, entry timestamp, draw block height, draw block hash, draw result (winner/non-winner), position trading history (list of prior owners with timestamps and prices). Winner PoolShare NFTs receive additional metadata: prize amount in sats, Lightning payment hash of prize payment, a special visual designation. Minting trigger: PoolShare NFTs are minted automatically at draw completion for all positions. Non-winners receive standard PoolShare. Winners receive Winner PoolShare with gold visual treatment. Transfer mechanics: PoolShares can be transferred/sold on any Ordinals-compatible marketplace. The underlying position is no longer active (draw has completed) but the NFT serves as a permanent verifiable record. A verification API allows any third party to verify the authenticity of any PoolShare NFT against the BitPoolz public ledger.

## Key Claims
- Bitcoin Ordinals-based NFT for pool position verification
- Standard metadata schema for position history
- Winner NFT with special designation and prize payment proof
- Automatic minting at draw completion for all positions
- Cross-platform transferability on Ordinals marketplaces
- Third-party verification API against public ledger
- Trading history embedded in NFT metadata
